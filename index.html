Videojuego

“Dicen que hay sucesos tan extraños que nadie puede explicar, empiezan como rumores para asustar a los niños, pero esta historia que murmuran los labios de los pueblos es una leyenda que puede trastornar a la gente.” 
En este proyecto se elabora un videojuego survival horror en 2D, realizando las distintas etapas del desarrollo del videojuego; desde el arte, las mecánicas y la programación de este.

Demo

Etapas del Proyecto

Etapa 1
Narrativa

Se realiza un texto narrativo que cuente la historia del videojuego, todo a forma de prosa. Se hace especial énfasis en las acciones de los personajes, el ambiente y las diferentes vertientes que puede tener la historia.

Etapa 2
Guion.

En esta etapa se pasa la narración a un guion de diálogos y acciones, se crea el árbol de diálogos que se utilizará en el videojuego.

Etapa 3
Escaleta y Mecánicas.

La escaleta define los escenarios, la libertad de movimiento que tendrán los personajes, así como los elementos con los que jugador podrá interactuar. También, se diseñan las mecánicas de juegos, a esto también se le conoce como (Game Design).

Communities

Aplicación Web dedicada a la administración, planificación de eventos y documentación de actividades culturales para las comunidades de la UPIICSA, Esta herramienta facilitará los diferentes procesos que conllevar realizar una feria, una exposición, un panel de expertos, etc.

Demo

Etapas del Proyecto

Etapa 1
Diseño de Requerimientos.

Se analizan los detalles de todos los procesos que conlleva realizar un evento cultural y basado en la investigación se destacan aquellos que pueden recibir una autorización virtual mediante una aplicación web detallando el desarrollo en diagramas y medios visuales.

Etapa 2
Maquetación y Diagramas

Los diagramas que se manejan varían dependiendo de las secciones que lleva el proyecto, estos diagramas y vistas tienen el objetivo de comunicar a los miembros del equipo a detalle todo lo necesario para el flujo de la información y la experiencia del usuario. Las tecnologías que se utilizan generalmente son trello y lucidChart (o a manopla).

Etapa 3
Desarrollo

Para la etapa de desarrollo se toma en cuenta el seguimiento de la planificación en trello junto a las representaciones de pantallas, de esta manera se crea de manera paralela en React.js la maquetación necesaria para los elementos que conforman este proyecto y también el backend que validara los elementos necesarios para que la aplicación tenga comunicación con la API y base de datos.

Sistema SmartBussines

El propósito de la herramienta es convertirse en un sistema de gestión de negocio completo, que integre algoritmos que resulten en cierto momento predictivos para algunos parámetros del negocio.

Demo

Etapas del Proyecto

Etapa 1
Documento de requerimientos

Analizar los requisitos que los dueños de negocio buscan para su negocio, se incluyen todos aquellos que podrían ser útiles para un negocio, se seleccionan los requisitos que son útiles para el sistema y también los requisitos que son necesarios para la realización técnica del sistema, dichos requisitos se plasman en un documento de requerimientos.

Etapa 2
Maquetación y diagramas

Una vez obtenido el documento de requerimientos, se procede a realizar un correcto diseño de los diagramas, para ello se utiliza la herramienta Trello

Etapa 3
Desarrollo

En esta etapa es necesario tener conocimientos básicos en java y/o javascript, se seguirá la planeación acordada en la plataforma trello, se realizarán las interfaces gráficas y el código del frontend de manera paralela, el lenguaje de programación que se utilizará para el frontend será java y xml para las respectivas interfaces de usuario, para el lenguaje de programación del backend será con javascript.
